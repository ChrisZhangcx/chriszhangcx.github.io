python & pytorch 安装指南

发表时间 2019-09-16   |   更新时间 2019-09-16

Here is a brief introduction to lead you to install both python & pytorch successfully.

If you found this page in a personal blog, plz note that this page will be deleted some weeks later.

1. python 3.7

Python是一门编程语言。要在本地运行python，需要事先安装python解释器（python interpreter）。

在此推荐安装 python 3.6.x / 3.7.x 版本（注意请务必安装python 3，如果已经安装了python 2的建议换成python 3），以下的操作示例以 python 3.7.4 为准。

1.1 installation

你可以通过以下两种方式来安装指定版本的python解释器：其中Anaconda比较简单方便，但是软件本身得有400~500M，比较大。直接下载python的话应该只会占用不到100M的空间。请根据自身需求决定。

1.1.1 使用Anaconda 3（推荐）

Anaconda是一个python版本与环境的管理软件。它被应用于这样的场景中：

考虑你有多个开发需求，其中一个是开发一个网页，需要flask-related modules；另外一个是写tensorflow model，需要tf-related modules；此外还有类似beatiful-soup-related的爬虫需求。你希望在做多个需求的时候，每个需求拥有独立的运行环境。一个方法是安装vitual-env module然后在命令行中进行管理，不过对于新手不够友好。

而Anaconda则允许你在本地以图形界面的形式创建多个隔离的运行环境。

Anaconda与python一样分为2与3两个版本，其中自动集成了对应版本的python。

点击此处下载Anaconda 3。

在网页中选择指定的系统（windows / mac / linux），然后选择Python 3.7 version即可。十分推荐下载64-Bit的版本，因为大家的电脑现在基本上都是64位的。

打开下载好的Anaconda3-****.exe文件，一路向下，如果没有说明，请不要修改其他默认设定，除非你具有修改设定的需求并知悉修改后可能造成的影响。

在 Choose Install Location 页面，如果C盘空间允许，强烈建议使用默认地址，亲测修改地址后会出现部分python module与所选的地址不匹配，在使用时找不到或者需要另外设定的情况（但是有时候乱搞一通可以解决）。

在 Advanced Installation Options 页面，把两个选项都勾上。第一个是允许Anaconda修改你的环境变量，这在安装某些module的时候十分有用；第二个是将Anaconda中自带的python作为你电脑中默认的python。

Anaconda的使用，请参照任意以 Anaconda 3 安装与使用 为关键词的百度搜索结果。

直接下载

在 这个地址 选择需要的python release version下载，注意选择你所需要的版本与平台。

1.2 double-check

下载完成后，WIN + R 输入 cmd 打开命令行，输入python并回车，如果出现 >>> 并等待你的进一步输入，则说明安装成功。在 >>> 下输入 exit() 退出交互式界面。

在命令行中输入 python -V 来查看电脑中所安装python的版本。如果安装成功，应当显示 Python 3.7.4 之类的结果。

使用Anaconda安装的python基本上不会有问题，如果直接下载的python无法完成上述检测，请确认：python37这个python安装目录已经被添加到环境变量中。

1.3 pip & conda

pip / conda 是module的管理工具。也就是由别人造出来的工具，像tensorflow / pytorch / keras / numpy都是python中的module，这些第三方module的功能涵盖但不限于：数值计算、爬虫、建模、网页开发与部署、运维、数据库。

如果你使用Anaconda安装了python，请将以下的 pip 关键字替换为 conda 。

在命令行中输入 pip list 来查看是否能够使用pip。如果不能，请确认pip.exe这个文件目录在环境变量中。这个文件一般会出现在python37文件夹的scripts文件夹中。

如果运行成功，将出现一系列的 module name [\t] version的结果。如果提示找不到，请与同学交流或百度如何解决。

2. pytorch (cpu version)

pytorch是一个建模工具，它比tensorflow封装的好一些，并且支持动态流图（tensorflow 2.0仍然是个弟弟，不建议使用）。封装更高的是keras，但是封装度太高导致很多优化没法做 另外我也不怎么会，所以我建议大家使用pytorch。

2.1 installation

在 这个网址 中来设定适合自己的pytorch版本并获得本地的安装命令。简单示例：

  • 如果你使用 windows，使用 Anaconda 安装了 python 3.7.x ，请在 命令行 中输入 conda install pytorch torchvision cpuonly -c pytorch
  • 如果你使用 windows，直接 安装了 python 3.7.x ，请在 命令行 中输入 pip3 install torch==1.2.0+cpu torchvision==0.4.0+cpu -f https://download.pytorch.org/whl/torch_stable.html。

在安装过程中会提示需要安装其他依赖库并要求确认，按要求输入y并回车就好。

注意，选项中的CUDA是一个GPU加速运算的模块，在不熟练（以及系统不支持）的情况下不建议大家使用，选择 None 即可。

2.2 double-check

确认pytorch安装完成：

  1. 使用 WIN + R 输入 cmd 打开命令行；
  2. 输入 python 进入 >>> 交互式界面；
  3. 输入 import torch 并回车，稍作等待，如果不报任何错误，直接进入下一行的 >>> ，则说明安装成功。
  4. 在3的基础上，输入 torch.__version__ 并回车，能够查看所安装的pytorch版本。

3. Notes

  • 如果有任何一个步骤出现问题，首先推荐百度解决方案。python和pytorch的安装已经有无数人踩坑并且写了blog，我写的肯定没有他们详细。换句话说，如果你采用网上的安装教程，只要能通过 1.2, 1.3, 2.1 三个步骤的检验，就说明安装没有问题。
